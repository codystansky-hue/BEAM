# BEAM — Composite Beam Cross-Section Analyzer

A desktop engineering tool for full end-to-end analysis of thin-walled composite beam cross-sections. Starting from a 2D Rhino geometry and a composite material layup, it drives a chain of external solvers to produce:

- A 6×6 cross-section stiffness matrix (*K*)
- Global beam deflections and rotations under combined axial, transverse, and thermal loads
- Euler (global) and CalculiX (local) buckling critical loads
- Interactive 3D visualization of deformed shapes and mode shapes

The UI is built with [Trame](https://kitware.github.io/trame/) (Vue3 + VTK.js) and runs as a local web server — open it in any browser.

## Prerequisites

### Python

Python **3.12** is required. The virtual environment bundled in `.venv/` is already configured for Python 3.12 on Windows.

If you need to recreate it:

```powershell
py -3.12 -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### Julia (required for global beam analysis)

1. Download and install Julia from **[julialang.org/downloads](https://julialang.org/downloads/)** — any 1.x release works.
2. Use the Windows installer and let it add `julia` to your `PATH`. Verify with `julia --version` in a new terminal.
3. Julia packages (`GXBeam`, `JSON`) are managed via `Project.toml` / `Manifest.toml` in the project root. On first run the solver script auto-installs them — **this requires internet access the first time**.

### CalculiX (required for local buckling analysis)

CalculiX is **not bundled** in this repository and can be hard to track down. You need the **Windows multi-threaded** (`ccx_MT.exe`) build.

**Where to get it:**

1. **Official site** — [www.calculix.de](http://www.calculix.de/) — links to community-maintained Windows packages under the *CalculiX on Windows* section.
2. **Pre-built GitHub releases** — search GitHub for `CalculiX Windows ccx_MT`. Community-maintained releases (e.g. from the `calculix` org) provide a ZIP with `ccx_MT.exe` and the required runtime DLLs alongside it. The version used during development was **2.23.0**.
3. The binary requires the Intel OpenMP runtime (`libiomp5md.dll`). If the solver crashes immediately, install the [Intel oneAPI redistributable](https://www.intel.com/content/www/us/en/developer/tools/oneapi/toolkits.html) or copy the DLL from the same ZIP.

**Recommended install layout** (matches the app's default path):

```
BEAM-paraview\
  CalculiX-Windows\
    bin\
      CalculiX-2.23.0-win-x64\
        bin\
          ccx_MT.exe
          libiomp5md.dll     ← keep DLLs alongside the exe
```

After downloading, set the CalculiX path in the app's **Solver Paths** drawer (see [Configuring Solver Paths](#configuring-solver-paths)).

### SwiftComp (optional)

SwiftComp is **not bundled** in this repository. The built-in **CLT solver** handles cross-section stiffness without SwiftComp. If you have a SwiftComp license, place the executable at the path configured in the app's solver settings.

## Workflow

The app has four pages, accessible from the left-hand navigation drawer.

### Page 1 — Pre-Processing

Set up the geometry and material stack before running any analysis.

#### Geometry

Upload a `.3dm` (Rhino) or `.dxf` file containing a single curve representing the beam cross-section mid-surface. The app generates a structured quad mesh by offsetting the curve inward by the total layup thickness.

Mesh controls:
- **Element size along curve** (mm): controls node spacing along the curve
- **Elements through thickness**: number of elements through the wall

#### Material Library

The app ships with a small built-in library. Expand each sub-tab to manage materials:

| Sub-tab | Contents |
|---------|----------|
| **Fibers** | Raw fiber constituent properties (moduli, strengths, CTE, density) |
| **Resins** | Matrix/resin properties |
| **Laminae** | Homogenized ply properties — generated by the Lamina Generator |
| **Layup** | Stacking sequence (list of ply name + angle pairs) |

#### Lamina Generator

Combines a fiber and a resin into a homogenized orthotropic ply:

1. Select **UD Fiber** or **Woven Fabric** mode.
2. Pick a fiber and resin from the library.
3. Set fiber volume fraction *V_f* (and weave geometry for woven).
4. Select **Homogenisation Method**: *Micromechanics (Built-in)* uses ROM/Chamis/Schapery (UD only); *SwiftComp Material SG* requires an external license (supports woven).
5. Click **Compute** — shows the resulting *E₁₁, E₂₂, G₁₂, ν₁₂, CTE*.
6. Enter a lamina name and fiber areal weight (FAW), then click **Save Lamina**.

#### Layup Design

Add plies from the laminae library to build a stacking sequence. Each ply has a name and an orientation angle (degrees). The total thickness is computed automatically from the individual ply thicknesses.

---

### Page 2 — Solution Setup

Configure loads and boundary conditions before running.

| Section | Description |
|---------|-------------|
| **Boundary Conditions** | Beam end conditions: Fixed-Free (cantilever), Pinned-Pinned, Fixed-Pinned, Fixed-Fixed |
| **Applied Load** | Resultant force magnitude (N) and angle from the beam axis (°) |
| **Thermal Loading** | Optional linear temperature gradient across the cross-section width |
| **Snippet Geometry** | Length (mm) and mesh density for the 3D CalculiX sub-model |

---

### Page 3 — Results

Run the analysis pipeline and view numerical results.

The pipeline has three independent stages that run in order. You can run all three at once or trigger them individually:

| Stage | Button | What it does |
|-------|--------|--------------|
| **1 — Mesh** | `1 · Mesh` | Reads the .3dm, generates the 2D quad mesh, assigns material properties |
| **2 — Global** | `2 · Global` | Computes 6×6 *K* matrix (CLT or SwiftComp) + GXBeam (deflections, Euler buckling) |
| **3 — Local** | `3 · Local` | Runs CalculiX (local buckling eigenvalue) |

Results displayed:
- **6×6 stiffness matrix** *K* (units: N, Nm, Nm²)
- **Tip deflections and rotations** (mm, rad)
- **Euler critical loads** *P_cr,22* and *P_cr,33* (N)
- **CalculiX buckling factor** (dimensionless multiplier on the applied load)

A scrollable **pipeline log** below the buttons shows timestamped output from each stage.

---

### Page 4 — Visualization

Interactive 3D viewport with results from the most recent solve.

| Control | Description |
|---------|-------------|
| **View selector** | Switch between: Cross-section mesh, 3D snippet (extruded), Deformed beam, Buckling mode |
| **Warp factor** | Scale the displacement for visibility (does not change the color scale) |
| **Scalar field** | Color the mesh by: Displacement magnitude, Von Mises stress, S11, S22, … |
| **Show edges** | Toggle element edge lines |
| **Show undeformed** | Overlay the original (undeformed) wireframe |
| **Snap view** | Jump to ±X, ±Y, ±Z orthographic views |
| **Reset camera** | Re-center and re-fit the camera to the current mesh |

Mouse controls:
- **Left drag**: Rotate
- **Middle drag / Shift+Left drag**: Pan
- **Scroll wheel**: Zoom

## Physical Modeling

### What you're analyzing

A **composite beam** (e.g. a deployable boom or structural tube) made from fiber-reinforced plies wrapped around a thin-walled cross-section. The goal is to predict how stiff it is, how much it deflects under load, and at what load it buckles — both globally (whole beam column) and locally (cross-section wall wrinkling).

### Stage 1 — Cross-Section Meshing

**Input:** A 2D curve (`.3dm` or `.dxf`) representing the mid-surface spine of the cross-section wall.

The spine curve is offset inward by the total layup thickness, creating inner and outer boundaries. A structured quad mesh (Q4 elements) is generated between them — this is the 2D cross-section finite element model. Each element gets a local material orientation from the tangent to the spine at that point, rotated by the user's ply angle.

**Physical meaning:** The cross-section is discretized so that a homogenization solver can integrate through it and compute effective beam stiffness properties.

### Stage 2a — Cross-Section Homogenization

The app offers two solvers (selectable in Solution Setup → Solver Settings):

- **CLT (Built-in)** — Classical Lamination Theory with Bredt–Batho torsion for closed thin-walled sections. No external dependencies.
- **SwiftComp** — Variational Asymptotic Beam Section (VABS) theory. Requires a licensed executable.

The output is the **6×6 beam stiffness matrix *K***:

```
[F1]   [K11 ...  K16] [ε1 ]
[F2]   [            ] [κ2 ]
[F3] = [            ] [κ3 ]
[M1]   [            ] [κ1 ]
[M2]   [            ] [κ2 ]
[M3]   [K61 ...  K66] [κ3 ]
```

F1 is axial force, F2/F3 are shear, M1 is torque, M2/M3 are bending moments; ε/κ are the corresponding strains/curvatures. The diagonal entries are the familiar EA, GA, GJ, EI₂₂, EI₃₃.

**Physical meaning:** The composite cross-section — with anisotropic plies at various angles — is reduced to a single point with 6 stiffness values. All ply-level mechanics are captured here.

### Stage 2b — Euler Buckling (Closed Form)

Uses the bending stiffnesses EI₂₂ and EI₃₃ extracted from *K*:

```
P_cr = π² EI / (K·L)²
```

where *K* is the end-condition factor (2.0 cantilever, 1.0 pinned-pinned, etc.) and *L* is the beam span. Two critical loads are computed — lateral (EI₂₂) and vertical (EI₃₃). This is **global column buckling** — the whole beam bending like Euler's classical column.

### Stage 2c — Global Beam Analysis (GXBeam)

GXBeam solves the **geometrically exact beam equations** (Hodges–Yu theory). Unlike Euler-Bernoulli, this handles large deflections, initial curvature, and full 6-DOF coupling. The 6×6 *K* matrix from SwiftComp is the constitutive law.

**Loads applied:**
- **Mechanical tip load** — decomposed into axial (Fx) and transverse (Fz) components from `load_angle_deg`
- **Thermal distributed loads** — if enabled, a spanwise temperature gradient is integrated over the cross-section elements to compute an equivalent axial force (F_th) and bending moments (M2_th, M3_th) from differential thermal expansion across the section width

**Output:** 6-component tip displacement/rotation vector (translations in mm, rotations in rad).

**Physical meaning:** Given the effective beam stiffness, this predicts how much the real structure deflects under the combined mechanical and thermal load field.

### Stage 3 — Local Buckling (CalculiX)

Euler buckling assumes the cross-section stays rigid. **Local buckling** is when the thin *walls* of the cross-section wrinkle — this typically governs thin-walled composite booms before global column buckling does.

The 2D cross-section mesh is **extruded** along Z into a 3D solid of length `snippet_length` — a short representative segment of the beam. A CalculiX `.inp` deck is written with:

- **C3D8 brick elements** (8-node hexahedra)
- Orthotropic material properties per element (in MPa)
- Boundary conditions (fixed root, free or pinned tip)
- Optional thermal pre-load step applying the temperature gradient to pre-stress the section
- A `*BUCKLE` step requesting the first eigenvalue

CalculiX runs a **linear perturbation buckling analysis** (Lanczos eigensolver): it computes the geometric stiffness matrix K_G from the applied compressive load, then solves `(K − λ K_G) φ = 0` for the smallest eigenvalue λ.

**Output:**
- **Buckling factor λ** — the applied load must be multiplied by λ to reach local buckling. If λ < 1, the section buckles under the current load.
- **Critical local load** = |applied load| × λ
- **Buckling mode shape** VTK — shows which wall segment deforms first

**Physical meaning:** You are asking: *at what compressive load does this thin-walled tube wall snap into a wrinkle pattern?* The thermal pre-stress step is significant for space structures — differential thermal expansion can substantially reduce local buckling capacity before any mechanical load is applied.

### The Full Picture

```
2D curve (.3dm / .dxf)
      │
      ▼  mesher.py
2D cross-section mesh (Q4)
      │
      ▼  CLT or SwiftComp
6×6 beam stiffness matrix K
      │
      ├──▶  Euler formula          →  Global P_cr  (column buckling)
      │
      ├──▶  GXBeam (Julia)         →  Tip deflections under load + thermal
      │
      └──▶  CalculiX (*BUCKLE)     →  Local P_cr   (wall wrinkling) + mode shape
```

The two buckling loads are compared — whichever is smaller governs the design. The thermal pre-load step in CalculiX can significantly reduce local buckling capacity in thermally loaded space structures.

Solver paths are configured in the **Solver Paths** panel at the bottom of the navigation drawer. Changes persist for the session.

|--------|-------------|-------|

## Troubleshooting

### App doesn't start / import error

Make sure dependencies are installed in the virtual environment:
```powershell
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### GXBeam stage fails: "Julia not found" or package errors

1. Confirm `julia` is on your PATH: open a terminal and run `julia --version`.
2. On first run, Julia downloads `GXBeam` and `JSON` — allow internet access.
3. If packages fail to install, open Julia manually and run:
   ```julia
   using Pkg
   Pkg.activate("path/to/BEAM-paraview")
   Pkg.instantiate()
   ```

### CalculiX stage fails: "executable not found"

Update the CalculiX path in the **Solver Paths** drawer panel. Use the full absolute path to `ccx_MT.exe`.

### Browser shows blank page or old content

Hard-refresh the browser (`Ctrl+Shift+R`) after the terminal confirms the server is ready.

### Warp factor changes cause the viewport to go black

This is fixed in the current version. If it occurs, click **Reset View** in the Visualization page to restore the camera.
